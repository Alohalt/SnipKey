# SnipKey
核心能力 使用流程 快速开始 相关链接
macOS 13+ · Swift 5.9+ · 开源

让常用文本输入，从重复劳动变成一次触发。

SnipKey 是一个面向 macOS 的菜单栏文本扩展工具。你只需要在任意应用中输入 #trigger，就能通过跟随光标的补全面板快速选择候选， 或直接展开为预设文本。

任意应用可用 系统范围监听输入，不局限于单一编辑器。
原生补全体验 候选面板跟随光标，支持键盘无缝确认。
内容可持续管理 三栏设置、变量插入、导入导出一体化。

核心交互预览

输入不中断

从触发、候选到展开，全程用键盘完成，不打断当前编辑流程。

内容更智能

动态变量、剪贴板建议、触发词校验，让常用文本可维护、可复用。

核心能力

把“触发、选择、展开、维护”做成一条顺滑链路。

页面内容按真实使用路径展开，先说明它能替你省下什么，再说明它如何在日常输入中稳定工作。

输入阶段：快速捕获触发词

系统范围监听键盘输入，捕获 # 前缀触发词，在任意可输入文本的应用中都能触发。

全局触发 菜单栏常驻 低打扰

选择阶段：补全面板跟随光标

浮动候选面板紧贴当前输入位置展示匹配结果，支持上下选择、Tab / Enter 确认、Esc 取消。

跟随光标 键盘驱动 频率排序

展开阶段：内容不只是静态文本

在替换内容中嵌入 {date}、{time}、 {clipboard}、{cursor} 等变量，让模板带上上下文。

动态变量 上下文插入 可编辑光标位

维护阶段：原生三栏设置界面

分组、搜索、创建、编辑、删除与 JSON 导入导出都在一个原生设置窗口内完成，适合持续积累常用片段。

三栏设置 搜索管理 JSON 导入导出

智能增强：剪贴板建议

记录剪贴板历史，重复复制内容时主动建议一键生成新的 Key，并在本地生成简短 trigger，帮助你把“临时复制”沉淀为“长期复用”。

剪贴板历史 智能建议 本地生成 trigger

稳定性：触发词校验与归一化

触发词仅允许字母、数字、下划线；保存时校验唯一性；导入历史数据时自动归一化，减少维护成本。

唯一性校验 自动归一化 更少冲突
使用流程

第一次配置后，后续使用只剩 4 个动作。

页面按照“先理解，再上手”的节奏组织信息，让新用户能快速建立使用心智。

1

打开设置

启动应用后，从菜单栏进入设置界面，开始维护自己的常用文本片段。

2

创建 Key

例如创建 account，并填写需要插入的替换内容或变量模板。

3

任意应用中触发

回到聊天、文档、邮件等输入场景，输入 #account 即可看到候选。

4

键盘确认展开

通过 Tab、Enter 或终止字符确认，SnipKey 会自动替换已输入触发词。

快速开始

推荐先用签名开发包运行，权限更稳定。

对新用户来说，最关键的信息是“怎么最快跑起来”，因此这里保留最短路径，并把更完整的说明延伸到仓库文档。

推荐方式

让 macOS 把辅助功能权限绑定到稳定的应用身份上，减少每次重建后重新授权的问题。

已签名开发包 权限更稳定 适合长期使用 
make signing-help
make bootstrap-personal-team
make run

# 如果本机已有可用 Apple Development 证书：
make test
make run

更完整的签名说明见 docs/development-signing.md
